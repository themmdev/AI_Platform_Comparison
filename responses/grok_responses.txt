"### Superposition: Being in Two States at Once

Superposition is weird for a single particle, while entanglement is extra weird because it links particles together like invisible string! Quantum stuff is super strange, but that's what makes it cool."

#include <cctype>

int countWordsStartingWithVowel(const std::string& str) {
    int count = 0;
    bool inWord = false;

    for (size_t i = 0; i < str.size(); ++i) {
        unsigned char ch = static_cast<unsigned char>(str[i]);
        if (isalpha(ch)) {
            if (!inWord) {
                char lower = tolower(ch);
                if (lower == 'a' || lower == 'e' || lower == 'i' || lower == 'o' || lower == 'u') {
                    count++;
                }
                inWord = true;
            }
        } else {
            inWord = false;
        }
    }
    return count;

"Here are a few clean and reliable SQL queries to find the **top 3 highest-paid employees** in the **Sales** department from a table named `Staff`.

### Most common and portable version (recommended)

```sql
SELECT 
    employee_id,
    first_name,
    last_name,
    department,
    salary
FROM Staff
WHERE department = 'Sales'
ORDER BY salary DESC
LIMIT 3;
```

### Alternative versions (depending on your database system)

#### 1. SQL Server / older versions (using TOP)

```sql
SELECT TOP 3
    employee_id,
    first_name,
    last_name,
    department,
    salary
FROM Staff
WHERE department = 'Sales'
ORDER BY salary DESC;
```

#### 2. Using ROW_NUMBER() — works in PostgreSQL, SQL Server, Oracle, etc. (very useful when you need ties handled explicitly)

```sql
SELECT *
FROM (
    SELECT 
        employee_id,
        first_name,
        last_name,
        department,
        salary,
        ROW_NUMBER() OVER (ORDER BY salary DESC) AS rank
    FROM Staff
    WHERE department = 'Sales'
) ranked
WHERE rank <= 3;
```

#### 3. Using DENSE_RANK() — if you want to include ties (e.g., if two people have the same top salary)

```sql
SELECT *
FROM (
    SELECT 
        employee_id,
        first_name,
        last_name,
        department,
        salary,
        DENSE_RANK() OVER (ORDER BY salary DESC) AS salary_rank
    FROM Staff
    WHERE department = 'Sales'
) ranked
WHERE salary_rank <= 3
ORDER BY salary DESC;
```

### Quick reference – which one to use

| Database          | Recommended approach          |
|-------------------|-------------------------------|
| MySQL / MariaDB   | `LIMIT 3`                     |
| PostgreSQL        | `LIMIT 3`                     |
| SQL Server        | `TOP 3` or `ROW_NUMBER()`     |
| Oracle (12c+)     | `FETCH FIRST 3 ROWS ONLY` or `ROW_NUMBER()` |
| SQLite            | `LIMIT 3`                     |

### Most universal and clean version (works almost everywhere)

```sql
SELECT 
    employee_id,
    first_name,
    last_name,
    department,
    salary
FROM Staff
WHERE department = 'Sales'
ORDER BY salary DESC
FETCH FIRST 3 ROWS ONLY;
```

This syntax is part of the SQL standard (supported in PostgreSQL, Oracle 12c+, SQL Server 2012+, etc.).

Pick the version that matches your database system — the simplest `ORDER BY ... LIMIT` version is usually the most readable and widely understood."

   - Not allowed: Submit AI-generated text, code, or answers as your own work without significant modification and citation.

3. **Always Do Your Own Thinking and Writing**  
   Treat AI output as a starting point. Read it, understand it, rephrase it in your own words, and combine it with your own ideas. Your final submission should reflect your personal effort and understanding.

4. **Cite AI When Required**  
   If your assignment allows AI assistance and requires sources:  
   - Cite the tool (e.g., "Explanation adapted from a response by Grok, xAI, December 2025").  
   - Some style guides (APA, MLA) now have formats for citing AI—follow them if instructed.

5. **Use AI to Check or Improve Your Work**  
   - Allowed: Run your own draft through AI to ask for feedback (e.g., "How can I improve this paragraph?" or "Check for grammar errors").  
   - Allowed: Ask AI to verify your math/solution steps after you've solved it yourself.

6. **Never Use AI on Closed-Book or Proctored Assessments**  
   Exams, quizzes, or timed tests usually prohibit all external tools, including AI. Using it here is almost always cheating.

7. **Prioritize Learning Over Grades**  
   The goal of homework is to help you learn. Relying too heavily on AI shortcuts defeats that purpose and leaves you unprepared for exams or real-world applications.

By using AI as a supportive learning tool—rather than a replacement for effort—you enhance your understanding while staying honest and ethical. When used wisely, AI can make you a better student."
